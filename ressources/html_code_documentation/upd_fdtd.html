upd_fdtd module¶

upd_fdtd.upd_p_fdtd_srl(p, p1, p2, fsrc, fsrc1, fsrc2, Nb, c, rho, Ts, dx, Cn, A, B, C, depth)[source]¶

This FDTD update calculates the pressure at the discrete time n+1 (p), as a function of the pressures at time n (p1) and n-1 (p2). The stencil follows a Standard RectiLinear (SRL) implementation. This update is implemented with a frequency-dependent boundary condition.

Parameters:
  • p (2D numpy array of floats.) – updated pressure at time n+1, (Pa).
  • p1 (2D numpy array of floats.) – current pressure at time n, (Pa).
  • p2 (2D numpy array of floats.) – past pressure at time n-1, (Pa).
  • fsrc (2D numpy array of floats.) – soft source at time n+1, (Pa).
  • fsrc1 (2D numpy array of floats.) – soft source at time n, (Pa).
  • fsrc2 (2D numpy array of floats.) – soft source at time n-1, (Pa).
  • Nb (2D numpy array of integers.) – boundary of the domain (1 if BC, 0 else).
  • c (float) – sound speed (m.s-1).
  • rho (float) – air density (kg.m-3).
  • Ts (float) – time step (s).
  • dx (float) – spatial step (m).
  • Cn (float) – Courant number.
  • A (float) – inertance of the boundary.
  • B (float) – stiffness of the boundary.
  • C (float) – resistivity of the boundary.
  • depth – index of depth of the velocity and pressure layer used for

the impedance implementation :type depth: int :return: the updated pressure at the time n+1, variable p. :rtype: 2D numpy array of floats.

upd_fdtd.upd_vel_pbc_fdtd(p, c, rho, Ts, dx, v_x, v_y, p_bc, p1_bc, K, a_k_cor, gamma_k, psi_k, psi1_k, depth)[source]¶

Velocity and pressure update defined for the impedance implementation. The velocity normal to the boundary is required for the definition of the ground impedance. The stencil follows a Standard RectiLinear (SRL) implementation.

Parameters:
  • p (2D numpy array of floats.) – updated pressure at time n+1, (Pa).
  • c (float) – sound speed (m.s-1).
  • rho (float) – air density (kg.m-3).
  • Ts (float) – time step (s).
  • dx (float) – spatial step (m).
  • v_x (2D numpy array of floats.) – velocity following the x-axis direction at n+1 (m.s-1)
  • v_y (2D numpy array of floats.) – velocity following the y-axis direction at n+1 (m.s-1)
  • p_bc (2D numpy array of floats.) – presure on the boundary at n+1 (Pa)
  • p1_bc (2D numpy array of floats.) – presure on the boundary at n (Pa)
  • K (int) – order of the recursive convolution method for the ground imp.
  • a_k_cor – Miki’s model correction for TDBC implementation
  • see [Guillaume_jsv2011, Sec.3.2.2] and get_imped_coefts.py, line 91.
Parameters:
  • gamma_k (list of floats) – poles of the partial fraction expansion
  • psi_k (2D numpy array of floats.) – impedance accumulator at time n+1 for the recursive convolut.
  • psi1_k (2D numpy array of floats.) – impedance accumulator at time n for the recursive convolut.
  • depth – index of depth of the velocity and pressure layer used

for the impedance implementation :type depth: int

Parameters:
  • v1_x (2D numpy array of floats.) – velocity following the x-axis direction at n (m.s-1)
  • v1_y (2D numpy array of floats.) – velocity following the y-axis direction at n (m.s-1)
Returns:

the updated pressure at the time n+1 - p ; the pressure

at the boundary - p_bc ; the velocity components - v_x and v_y ; and the accumulator - psi_k. :rtype: 2D numpy array of floats.
